# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Argent, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, email **hola@zenemig.net** with:

- A description of the vulnerability
- Steps to reproduce it
- The potential impact
- Any suggested fix (optional)

## Response Timeline

- **Acknowledgment:** Within 48 hours of your report.
- **Assessment:** Within 1 week, we'll confirm the vulnerability and its severity.
- **Fix:** We aim to release a patch within 2 weeks of confirmation, depending on complexity.

## Scope

This policy covers the Argent application code in this repository. For vulnerabilities in third-party dependencies, please report them to the respective maintainers and open an issue here so we can update our dependency.

For Supabase infrastructure issues, report directly to [Supabase Security](https://supabase.com/security).

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest on `main` | Yes |
| Older versions | No |

## Credit

We're happy to credit security researchers in our release notes (with your permission).

Thank you for helping keep Argent and its users safe.
